# Конструктор шаблона паспорта изделия

## Обзор

Теперь вы можете настраивать структуру и содержание паспорта изделия через веб-интерфейс. Система позволяет:
- Добавлять, редактировать и удалять разделы паспорта
- Изменять порядок разделов
- Включать/отключать отображение разделов
- Настраивать содержимое каждого раздела с использованием переменных

## Как использовать

### 1. Применение миграции БД

Перед началом работы примените SQL-миграцию:

```bash
mysql -u user -p database < sql/migrations/003_passport_template.sql
```

### 2. Доступ к конструктору шаблонов

1. Откройте страницу управления паспортами
2. Перейдите в раздел "Настройка шаблонов"
3. Вы увидите список всех разделов паспорта

### 3. Редактирование раздела

Для редактирования раздела:
1. Нажмите кнопку "Редактировать" рядом с нужным разделом
2. Измените:
   - **Заголовок** - название раздела (например, "ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ")
   - **Содержимое** - HTML-шаблон с переменными
   - **Порядок** - числовое значение для сортировки
   - **Активен** - показывать ли раздел в паспорте
3. Нажмите "Сохранить"

### 4. Переменные в шаблонах

В содержимом раздела можно использовать следующие переменные:

| Переменная | Описание | Пример значения |
|------------|----------|-----------------|
| `{product_name}` | Наименование изделия | "Двигатель асинхронный 5АМХ80А2" |
| `{product_model}` | Модель/артикул | "5АМХ80А2" |
| `{serial_number}` | Серийный номер | "SN-2401-12345-0001" |
| `{manufacture_date}` | Дата изготовления | "15.01.2024" |
| `{warranty_period}` | Гарантийный срок | "24 месяцев" |
| `{warranty_start}` | Начало гарантии | "15.01.2024" |
| `{warranty_end}` | Окончание гарантии | "15.01.2026" |
| `{org_name}` | Название организации | "ОАО «Полесьеэлектромаш»" |
| `{org_address}` | Адрес организации | "225644, Брестская область, г. Лунинец, ул. Красная, 179" |
| `{org_phone}` | Телефон организации | "+375 1647 2-78-09" |
| `{org_email}` | Email организации | "polesie@polesieelectromash.by" |

### 5. Специальные плейсхолдеры

#### Таблица спецификаций
Для раздела технических характеристик используйте плейсхолдер `{specs_rows}`, который будет автоматически заменён на строки таблицы со спецификациями изделия.

Пример шаблона для раздела спецификаций:
```html
<table border="1" cellpadding="5" cellspacing="0" width="100%">
    <tr>
        <th>Параметр</th>
        <th>Значение</th>
        <th>Ед. изм.</th>
    </tr>
    {specs_rows}
</table>
```

### 6. Стандартные разделы

По умолчанию система создаёт следующие разделы:

1. **ПАСПОРТ** (header) - заголовок документа
2. **СВЕДЕНИЯ ОБ ИЗГОТОВИТЕЛЕ** (manufacturer) - информация об организации
3. **ОСНОВНЫЕ СВЕДЕНИЯ ОБ ИЗДЕЛИИ** (basic_info) - наименование, модель, серийный номер
4. **ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ** (specs) - таблица параметров
5. **ГАРАНТИЙНЫЕ ОБЯЗАТЕЛЬСТВА** (warranty) - условия гарантии
6. **ТРЕБОВАНИЯ БЕЗОПАСНОСТИ** (safety) - меры предосторожности
7. **УСЛОВИЯ ХРАНЕНИЯ И ТРАНСПОРТИРОВКИ** (storage) - правила хранения
8. **СВИДЕТЕЛЬСТВО О ПРИЕМКЕ** (acceptance) - подпись ОТК

### 7. API методы

#### Получение шаблонов
```
GET /modules/production/api_passport.php?action=get_templates
```

Ответ:
```json
{
    "success": true,
    "templates": [
        {
            "id": 1,
            "section_key": "header",
            "title": "ПАСПОРТ",
            "content_template": "<h2>{product_name}</h2>",
            "sort_order": 1,
            "is_active": 1
        }
    ]
}
```

#### Сохранение шаблона
```
POST /modules/production/api_passport.php?action=save_template
```

Параметры:
- `section_key` - уникальный ключ раздела
- `title` - заголовок
- `content_template` - HTML-шаблон
- `sort_order` - порядок сортировки
- `is_active` - 1 или 0

## Примеры использования

### Пример 1: Добавление нового раздела "Сервисное обслуживание"

1. Создайте новую запись в таблице `passport_templates`:
```sql
INSERT INTO passport_templates 
(section_key, title, content_template, sort_order, is_active) 
VALUES 
('service', 'СЕРВИСНОЕ ОБСЛУЖИВАНИЕ', 
'<p>Техническое обслуживание проводится каждые 12 месяцев.</p>
<p>Контакт сервисной службы: {org_phone}</p>', 
9, 1);
```

### Пример 2: Изменение текста гарантийного раздела

Отредактируйте шаблон раздела "warranty" через интерфейс или напрямую в БД:
```html
<p>Гарантийный срок: <strong>{warranty_period}</strong></p>
<p>Гарантия начинается: {warranty_start}</p>
<p>Гарантия заканчивается: {warranty_end}</p>
<p>Производитель: {org_name}</p>
```

### Пример 3: Скрытие раздела

Установите `is_active = 0` для раздела, который не нужно показывать в паспорте.

## Важные замечания

1. **Не удаляйте стандартные ключи разделов** (header, manufacturer, basic_info и т.д.) - они используются системой
2. **Проверяйте HTML-код** в шаблонах на корректность
3. **Все переменные обязательны** - если какая-то переменная не найдена, она будет заменена на пустую строку
4. **Порядок разделов** определяется полем `sort_order` (по возрастанию)
5. **Изменения применяются сразу** после сохранения и влияют на все новые паспорта
