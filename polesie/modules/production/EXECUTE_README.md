# Модуль исполнения производства

## 📋 Обзор

Страница `execute.php` - полнофункциональный интерфейс для управления производственными заданиями, фиксации производства товаров, списания материалов и контроля этапов производства.

## 🎯 Основные функции

### 1. Список активных заданий
- Отображение всех заданий со статусами "Планируется" и "В работе"
- Сортировка по приоритету (Срочный → Высокий → Нормальный → Низкий)
- Прогресс выполнения в реальном времени
- Быстрый выбор задания из боковой панели

### 2. Управление этапами производства
- Просмотр всех этапов для выбранного задания
- Статусы этапов: Ожидает, В работе, Завершено, Пропущено
- Кнопки "Начать" и "Завершить" для каждого этапа
- Автоматическое резервирование материалов при начале этапа
- Фиксация количества прошедших и забракованных единиц

### 3. Управление материалами
- Таблица всех требуемых материалов
- Индикация доступности: ✓ Достаточно, ⚠ Частично, ✗ Недостаточно
- Кнопка списания материалов со склада
- Автоматическое создание записей о движении материалов
- Проверка наличия на складе перед списанием

### 4. Серийные номера
- Генерация уникальных серийных номеров
- Привязка к продукту и производственному заданию
- Автоматическое создание при завершении производства
- Отображение всех созданных номеров

### 5. Завершение производства
- Модальное окно с указанием количества годных и брака
- Автоматическое создание серийных номеров (опционально)
- Автоматическое списание материалов
- Обновление статуса позиции заказа
- Фиксация даты завершения

### 6. Информация о задании
- Назначение исполнителя
- Изменение приоритета
- Добавление заметок
- Просмотр плановых и фактических дат

## 🗄️ Структура БД

### Новые таблицы

#### `material_movements` - Движения материалов
```sql
CREATE TABLE `material_movements` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `material_id` INT NOT NULL,
  `movement_type` ENUM('receipt', 'consumption', 'adjustment', 'return', 'transfer'),
  `quantity` DECIMAL(15,3) NOT NULL,
  `reference_type` VARCHAR(50),
  `reference_id` INT,
  `user_id` INT,
  `notes` TEXT,
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Существующие используемые таблицы
- `production_tasks` - Производственные задания
- `production_task_stages` - Этапы выполнения заданий
- `production_tasks_materials` - Материалы для заданий
- `product_serial_numbers` - Серийные номера продукции
- `materials` - Материалы (склад)
- `order_items` - Позиции заказов

## 🔗 API Endpoints

### `api_execute.php`

| Действие | Описание | Параметры |
|----------|----------|-----------|
| `start_stage` | Начало этапа | `stage_id`, `task_id` |
| `complete_stage` | Завершение этапа | `stage_id`, `task_id`, `quantity_passed`, `quantity_rejected` |
| `consume_materials` | Списание материалов | `task_id` |
| `create_serial` | Создание серийного номера | `task_id`, `product_id`, `serial_number` |
| `complete_production` | Завершение производства | `task_id`, `quantity_good`, `quantity_defect`, `auto_serial` |
| `update_worker` | Обновление исполнителя | `task_id`, `worker_id` |
| `update_priority` | Обновление приоритета | `task_id`, `priority` |
| `save_notes` | Сохранение заметок | `task_id`, `notes` |

## 💡 Рабочий процесс

### Типовой сценарий использования:

1. **Выбор задания**
   - Пользователь выбирает задание из списка слева
   - Отображается детальная информация о задании

2. **Начало производства**
   - Нажатие кнопки "▶ Начать" на первом этапе
   - Автоматическое резервирование материалов
   - Статус задания меняется на "В работе"

3. **Прохождение этапов**
   - Последовательное прохождение всех этапов
   - Фиксация количества прошедших контроль единиц
   - Учет брака на каждом этапе

4. **Списание материалов**
   - После начала производства можно списать материалы
   - Проверка наличия на складе
   - Создание записи о движении материала

5. **Завершение производства**
   - Нажатие кнопки "✅ Завершить производство"
   - Ввод количества годных и брака
   - Опциональное создание серийных номеров
   - Автоматическое списание материалов
   - Обновление статуса заказа

## 🔐 Права доступа

Требуемые права:
- `production.execute` - Исполнение производства
- `production.edit` - Редактирование заданий

Права назначаются ролям:
- **Технолог**: полный доступ к исполнению
- **Рабочий**: чтение и исполнение заданий

## 🎨 UI Компоненты

### Карточки статистики
- План - плановое количество
- Факт - фактически произведено
- Годные - прошедшие контроль качества
- Брак - забракованная продукция

### Вкладки рабочей области
1. **Этапы** - сетка карточек этапов
2. **Материалы** - таблица с доступностью
3. **Серийные номера** - список созданных номеров
4. **Информация** - детали задания

### Индикация статусов
- **Приоритеты**: Срочно (красный), Высокий (оранжевый), Нормальный (синий), Низкий (серый)
- **Этапы**: Ожидает (серый), В работе (голубой), Завершено (зеленый)
- **Материалы**: Достаточно (зеленый), Частично (желтый), Недостаточно (красный)

## 📊 Отчетность

Все действия логируются в:
- `material_movements` - движения материалов
- `production_task_stages` - история этапов
- `production_tasks` - история изменений задания
- `product_serial_numbers` - созданные серийные номера
