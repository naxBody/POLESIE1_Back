# Система ролей и прав доступа

## Обзор

В системе реализована гибкая система разделения прав доступа на основе ролей. Каждая роль имеет доступ только к тем модулям и функциям, которые необходимы для выполнения её задач.

## Роли пользователей

### 1. Администратор (`admin`)
- **Логин/пароль:** `admin / admin123`
- **Описание:** Полный доступ ко всем модулям системы
- **Доступные модули:** Все без ограничений
- **Права:** Просмотр, создание, редактирование, удаление во всех разделах

### 2. Директор (`director`)
- **Логин/пароль:** `director / director123`
- **Описание:** Руководство предприятия, просмотр всех отчетов
- **Доступные модули:**
  - ✅ Панель управления (просмотр)
  - ✅ Заказы (просмотр, редактирование)
  - ✅ Контрагенты (просмотр)
  - ✅ Производство (просмотр)
  - ✅ Продукция (просмотр)
  - ✅ Склад (просмотр)
  - ✅ Материалы (просмотр)
  - ✅ Сотрудники (просмотр)
  - ✅ Контроль качества (просмотр)
  - ✅ Отчеты (просмотр, экспорт)

### 3. Менеджер по продажам (`sales_manager`)
- **Логин/пароль:** `ivanov / manager123`
- **Описание:** Работа с заказами и клиентами
- **Доступные модули:**
  - ✅ Панель управления (просмотр)
  - ✅ Заказы (просмотр, создание, редактирование)
  - ✅ Контрагенты (просмотр, создание, редактирование)
  - ✅ Продукция (просмотр)
  - ✅ Производство (просмотр)

### 4. Технолог (`technologist`)
- **Логин/пароль:** `petrov / tech123`
- **Описание:** Управление производственными процессами
- **Доступные модули:**
  - ✅ Панель управления (просмотр)
  - ✅ Производство (просмотр, создание, редактирование)

- **Логин/пароль:** `sidorov / store123`
- **Описание:** Учет материалов и продукции на складе
- **Доступные модули:**
  - ✅ Панель управления (просмотр)

## Установка

### Шаг 1: Выполните SQL скрипт

```bash
mysql -u root -p polesie_production < sql/setup_roles.sql
```

Или выполните содержимое файла `sql/setup_roles.sql` через phpMyAdmin или другой MySQL клиент.

### Шаг 2: Проверка установки

Проверьте, что роли созданы:

```sql
SELECT * FROM user_roles;
```

Проверьте права доступа:

```sql
SELECT r.name as role_name, rmp.module, rmp.can_view, rmp.can_create, rmp.can_edit, rmp.can_delete
FROM user_roles r
JOIN role_module_permissions rmp ON r.id = rmp.role_id
ORDER BY r.name, rmp.module;
```

Проверьте пользователей:

```sql
SELECT u.username, u.full_name, r.name as role_name
FROM users u
JOIN user_roles r ON u.role_id = r.id;
```

## Как это работает

### В боковой панели (sidebar.php)

Меню автоматически адаптируется под роль пользователя:

```php
// Получаем доступные модули
$availableModules = getAvailableModules();

// Показываем только доступные разделы
<?php if (in_array('warehouse', $availableModules)): ?>
    <!-- Раздел Склад -->
<?php endif; ?>
```

### Проверка прав в коде

```php
// Проверка доступа к модулю
if (!canAccessModule('warehouse')) {
    redirect(pageUrl('index.php'));
}

// Проверка права на создание
<?php if (canCreateInModule('orders')): ?>
    <a href="create.php">Создать заказ</a>
<?php endif; ?>

// Проверка права на редактирование
<?php if (canEditInModule('production')): ?>
    <button>Редактировать</button>
<?php endif; ?>
```

## Примеры использования

### Для кладовщика

После авторизации как `sidorov / store123`:
- Видит только разделы: **Панель управления**, **Склад**
- Может: просматривать материалы, создавать поступления, редактировать остатки
- Не видит: Заказы, Производство (планирование), Контрагенты, Сотрудники

### Для рабочего

После авторизации как `worker1 / worker123`:
- Видит только разделы: **Панель управления**, **Производство**
- Может: просматривать задания, обновлять статус выполнения
- Не видит: Склад (операции), Заказы, Контрагенты

### Для менеджера

После авторизации как `ivanov / manager123`:
- Видит разделы: **Панель управления**, **Заказы**, **Контрагенты**, **Продукция**
- Может: создавать заказы, редактировать контрагентов
- Не видит: Склад (операции), Контроль качества

## Добавление новой роли

1. Добавьте роль в таблицу `user_roles`:

```sql
INSERT INTO user_roles (name, code, description, permissions) VALUES
('Новая роль', 'new_role', 'Описание', '{}');
```

2. Настройте права по модулям:

```sql
SET @role_id = LAST_INSERT_ID();

INSERT INTO role_module_permissions (role_id, module, can_view, can_create, can_edit, can_delete) VALUES
(@role_id, 'orders', TRUE, TRUE, FALSE, FALSE),
(@role_id, 'warehouse', FALSE, FALSE, FALSE, FALSE);
```

3. Создайте пользователя с новой ролью:

```sql
INSERT INTO users (username, password_hash, full_name, role_id) VALUES
('newuser', 'password123', 'Новый Пользователь', @role_id);
```

## Структура базы данных

### Таблица `user_roles`
- `id` - ID роли
- `name` - Название роли
- `code` - Код роли (уникальный)
- `description` - Описание
- `permissions` - JSON с правами
- `is_active` - Активность роли

### Таблица `role_module_permissions`
- `id` - ID записи
- `role_id` - Ссылка на роль
- `module` - Код модуля (dashboard, orders, warehouse, etc.)
- `can_view` - Право просмотра
- `can_create` - Право создания
- `can_edit` - Право редактирования
- `can_delete` - Право удаления

### Таблица `users`
- `id` - ID пользователя
- `username` - Логин
- `password_hash` - Пароль
- `full_name` - ФИО
- `role_id` - Ссылка на роль
- `is_active` - Активность пользователя

## Функции для работы с правами

| Функция | Описание |
|---------|----------|
| `canAccessModule($module)` | Проверка доступа к модулю |
| `canCreateInModule($module)` | Проверка права на создание |
| `canEditInModule($module)` | Проверка права на редактирование |
| `canDeleteInModule($module)` | Проверка права на удаление |
| `getAvailableModules()` | Получение списка доступных модулей |
| `requireModuleAccess($module)` | Перенаправление при отсутствии доступа |

## Безопасность

- Все проверки прав выполняются на сервере
- Боковая панель скрывает недоступные разделы
- При попытке прямого доступа к URL без прав происходит перенаправление
- Администратор имеет полный доступ ко всем функциям
